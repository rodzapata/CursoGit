    Configuracion inicial de Git

    git config --global user.name "Rodney Zapata"

    git config --global user.email "rodneyzapatapalacio@gmail.com"

    git config --global core.editor "code --wait"

    git config --global -e

    Configuracion del Enter en Windows

    git config --global core.autocrlf true

    Configuracion del Enter en Linux y Mac

    git config --global core.autocrlf input

    En las versiones anteriores la carpeta por defecto del repositorio se llamaba Master

    A parir de las versiones nuevas se llama main

    si trabajamos con una version vieja y queremos que por defecto tenga el nuevo nombre debemos escribir

    git config --global init.defaultbranch main

    en caso que ya veniamos trabajando con la carpeta master

    podemos renombrarla de master a main

    con el siguiente comando:

    git branch -m main

    repositorio de prueba: https://github.com/rodzapata/CursoPronunciation.git

    Inicializar el proyecto con Init

    Esto solo se hace cuando se va a crear el proyecto desde Cero

    git init


    podemos especificar la rama que vamos a clonar, por ejemplo como el repositorio

    por defecto es ahora main, pero yo venia trabajando con la version master, entonces tengo que especificar la rama con el paramatro -b

    de breanch que significa rama

    como se muestra a continuacion:

    git init -b master

    Eliminar un repositorio Git creado por error con git init

    para eliminar la carpeta de .git, que se creo en la carpeta errada, entonces escribimos

    rm -rf .git

    donde rm, es el comando para eliminar archivo, r, para que elimine de manera recursiva, osea incluyendo subdirectorio,

    y la f, para que force la eliminacion y no actualice la verificacion

    Comandos de transacciones

    Agregamos un archivo al area de Stage (Escenario)

    git add index.html

    Verificamos el estado del repositorio Status

    git status

    Confirmamos los cambios realizado con Commit

    git commit -m "Plantilla inicial"

    ⏫

    cancelar modificaciones ---nuevo metodo

    git restore index.html

    Descartar una modificacion que aun no haya utilizado Git add sin Commit

    git restore --staged index.html

    git status

    git restore index.html

    ⏫

    cancelar modificaciones ---metodo anterior por compatibilidad

    git checkout --index.html

    ⏫

    Cancelar el ultimo Commit realizado

    git reset HEAD index.html

    Log para ver los commit realizados para eliminar el commit

    git log --oneline

    Resetear un Commit realizado

    git reset 6048f11

    git checkout --index.html

    El comando Reset no es recomendado porque es destructivo es mejor utilizar Revert

    Excluir archivos que no queremos que se le haga versionamiento

    Hay archivo que no queremos que se se les haga copia como por ejemplo los ejecutables

    variables de entorno que son diferente en cada máquina, como credenciales de las bases de datos,

    que no queremos que suban en la copia por seguridad

    creacion del archivo .gitignore

    En este archivo colocamos los nombres y directorios que queremos excluir

    por ejemplo el archivo .env, donde tenemos las variables de conexion a Mysql

    git checkout -b Beta

    git checkout master

    Para actualizar la rama principal con los cambios realizado en la rama Beta

    Subir el codigo local a la nuve

    Hay que tener una cuenta en Gihub

    para subir el trabajo local a la nueve hay que utilizar el comando Remote

    git remote add origin https://github.com/rodzapata/CursoGit.git

    Cargamos los datos con el comando Push

    git push -u origin master

    pedira la cuenta de usuario y el token

    el parametro -u es para que cree una rama nueva en este caso master en el servidor remoto

    en el caso que la rama exista se debe omitir

    ⏫

    token Git

    Como cada cierto tiempo hay que generar un token hay que ralizar los siguientes pasos:
    dentro del menu que aparece en la parte superior derecha aparece un mundo con una cruz,
    escogemos settings y esogemos la ultima opcion Developer settings,
    por ultimo escogemos Personal access Token, le damos al boton Create new token
    debemos escribir una note un nombre por ejemplo Compaq28082022 para identificar,
    la maquina de donde se esta actualizando y la fecha de la actualizacion, ahora en Select scope
    hay que chulear la opcion repo (full control of private repositories),
    ojo que si no se chulea esta opcion no funciona el token, por utlimo le datos click al boton,
    Generate Token
    ⏫

    clonar el repositorio para trabajar y actualizara los datos

    para clonar el respositorio del remoto a nuestra maquina local se usa el comando Clone

    git clone https://github.com/rodzapata/CursoGit.git

    se recomienda que una vez descargados los cambios, se ejecute el comando Pull

    git pull

    realizamos los cambios a trabajar hacemos los Commit y finalamente actualizamos al remoto usando Push

    git push origin master

    Como saber si nuestro repositorio local esta conectado a uno remoto

    git remote -v

    En caso de haber creado un remoto equivoca se puede borrar y crearlo nuevamente, para borrarlo se unsa el comando

    git remote remove origin

    Como modificar la descripcion de un Commit

    Prueba de una nueva leyenda

    Para modificar la descripcion del commit, se escribe el comando git commit --amend

    Este comando funciona, si el commit es local, osea que nos hecho un push, en caso de haberlo hecho

    hay que forsar la actualizacion al contenedor remoto con el comando git push --force
